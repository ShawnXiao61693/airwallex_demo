# 周报模块 · 开发文档

> 给后续负责**部署 / 接线**的 agent。目标：把"手搓上传的周报 HTML"挂到线上，能通过稳定 URL 打开。
> 当前交付物：`web/weekly.html`（一份做好的第 26 周周报成品）。

---

## 1. 这个模块在架构里是什么

看仓库根 `README.md` 的架构图与 `engine/README.md`，周报**不走采集引擎**：

```
手搓上传 周/月报 HTML  →  publication（周/月·HTML）  →  审核发布  →  分发  →  用户端
```

> engine/README 原话：「**周报/月报：不走本引擎，手搓上传。**」

所以"周报模块"的本质就是：**人做好一份 HTML → 上传 → 线上能打开**。日报是引擎自动跑，周报是人工产出物的发布通道。

## 2. 当前交付物：`web/weekly.html`

- **形态**：5 页翻页"学习杂志"。封面 → 竞品 Battle Card（XTransfer）→ 异议处理（万里汇费率战）→ 新窗口获客（SHEIN 韩国站）→ 本周自测 + 学习榜。
- **依赖**：只依赖 `web/assets/style.css`（复用日报同一套设计系统）。**不依赖 `data.js`，无其他外链脚本**（字体走 Google Fonts CDN）。
- **交互**（纯前端，无后端）：底部翻页 / 页码圆点 / 键盘左右方向键；话术「复制」按钮（`navigator.clipboard`）；自测选择判分 + 积分联动学习榜。
- **占位**：AI 播客、短视频是**形态占位**，非真实媒体；页面已标注「⚙ 占位」。
- **内容来源诚实说明**：3 条选题取自 `web/assets/data.js` 第 26 周原料池，文案为人工改写。**这些新闻的真伪与具体数字（如 XTransfer 估值、万里汇费率）未经独立核实**，正式对外前建议复核。

⚠️ **路径敏感**：`weekly.html` 用相对路径引 `assets/style.css`。线上托管时必须保证从 `weekly.html` 同级能取到 `assets/`，否则样式 404（见 §6 坑）。

## 4. 实现路径

### 路线 A —— MVP：静态托管（推荐先做，最快让线上能开）

把 `web/` 作为静态资源挂到 `clawn.dev` 的一个路径或子域。

1. **上传文件到服务器**（任选其一）：
   - `git clone`/`pull` 本仓库到 `~/airwallex-demo`（推荐，后续更新只需 `git pull`）；或
   - `scp -i <key> -r web/ ubuntu@47.129.174.5:~/airwallex-demo/web`
2. **Nginx 加一段 location**（示例，路径方案）：
   ```nginx
   location /airwallex/ {
       alias /home/ubuntu/airwallex-demo/web/;
       index index.html;
       try_files $uri $uri/ =404;
   }
   ```
   或独立子域 `awx.clawn.dev`（需加 DNS + Cloudflare 记录 + server 块）。
3. `sudo nginx -t && sudo systemctl reload nginx`
4. **验收**：浏览器开 `https://clawn.dev/airwallex/weekly.html`，确认 5 页翻页、复制、自测都正常，且 `assets/style.css` 不 404。

> 这套方案下，"上传一份新周报" = 把新的 `weekly_YYYY-Wxx.html` 放进 `web/`（或专门的 `web/weekly/` 目录）+ `git pull`，即时可访问。简单，但已满足"上传 HTML、线上能打开"。

### 路线 B —— 完整周报模块：上传 + 发布（对齐架构 publication）

在 Flask 里把"手搓上传"做成真正的功能（对应架构的 `publication` 节点）。

- **存储**：`publications/` 目录存 HTML + 一个 `index.json`（或架构里的 `publication` 表）记录元数据：
  ```json
  { "slug": "2026-w26", "type": "weekly", "title": "本周三件最值得练的事",
    "publish_date": "2026-06-29", "file": "2026-w26.html", "status": "published" }
  ```
- **路由**（建议挂在 `/airwallex` 命名空间下）：
  | 方法 | 路径 | 作用 |
  |---|---|---|
  | POST | `/airwallex/admin/publications` | multipart 接收 HTML + 元数据 → 存盘 + 写 index |
  | GET | `/airwallex/admin/publications` | 列表（供 admin 审核页） |
  | GET | `/airwallex/weekly/<slug>` | 返回该周报 HTML |
- **鉴权**：admin 上传需简单口令 / 登录（demo 级即可）。
- **前端接线**：
  - `web/admin.html` 「周报审核 → 确认发布」按钮 → 调上传/发布接口；
  - `web/index.html` 加周报入口、`web/app.html` 周报 tab 链到 `/airwallex/weekly/<最新>`。
- **assets 同源**：上传的 HTML 仍引 `assets/style.css`，需保证服务时同源能取到（把 `assets/` 也挂在 `/airwallex/assets/`，或上传时把样式改成绝对路径 / 内联）。

### 推荐顺序
**先 A**（满足当前"线上能打开"的需求），再按 **B** 把上传发布补成产品形态。

## 6. 已知坑

- **assets 相对路径**：周报不放在能访问到 `assets/` 的同级位置 → `style.css` 404、页面无样式。用正确的 Nginx `alias/root`，或把样式改绝对路径 / 内联。
- **`file://` 剪贴板**：本地双击打开（`file://`）时 `navigator.clipboard` 受限，"复制"按钮无提示；走 `http(s)` 正常。
- **SSH IP 白名单**：部署方 IP 不在白名单则连不上，先放行或换到 Shawn 本机执行。
- **Cloudflare 缓存**：更新后若没刷新，可能需在 Cloudflare 清缓存 / 用 `cf-cache-status` 确认。
- **内容真伪**：`data.js` 第 26 周新闻未独立核实，对外前复核。
