# 报告上传接口（给制作/上传 agent）—— 周报 / 月报

把人工做好的**周报 / 月报** HTML 上传到平台，它会作为**草稿**进入后台对应审核页（周报审核 / 月报审核），由 RevE 预览后点「通过发布」。日报是 AI 多候选模型，不走这里。

- 服务器：`http://13.214.205.219`
- 鉴权 token：在服务器 `~/airwallex/engine/.env` 的 `API_TOKEN`（或找 Shawn 要）

---

## 1. 上传报告 → 草稿

```
POST http://13.214.205.219/api/reports
Header: X-Api-Token: <API_TOKEN>
Content-Type: multipart/form-data
```

| 字段 | 必填 | 说明 |
|---|---|---|
| `type` | 是 | `weekly`（周报）或 `monthly`（月报）|
| `period` | 是 | 期数标识 + 发布日 `YYYY-MM-DD`：周报用**当周周一**，月报用**当月 1 号** |
| `title` | 否 | 标题（后台列表显示用）|
| `file` | 是 | 报告 HTML 文件 |

**可用 period（2026 年 6–7 月）**：
- 周报（每周一）：`2026-06-01, 06-08, 06-15, 06-22, 06-29, 07-06, 07-13, 07-20, 07-27`
- 月报（每月 1 号）：`2026-06-01, 2026-07-01`

**curl 示例（月报）**：
```bash
curl -X POST http://13.214.205.219/api/reports \
  -H "X-Api-Token: <API_TOKEN>" \
  -F "type=monthly" \
  -F "period=2026-06-01" \
  -F "title=6月月报特刊·跨境支付的地壳动了" \
  -F "file=@monthly_2026-06.html"
```

**python 示例（周报）**：
```python
import requests
requests.post("http://13.214.205.219/api/reports",
  headers={"X-Api-Token": TOKEN},
  data={"type":"weekly","period":"2026-06-22","title":"第26周·费率战来了"},
  files={"file": open("weekly.html","rb")})
```

**响应**：`{"ok":true,"type":"monthly","period":"2026-06-01","status":"draft","url":"/airwallex/data/monthly/2026-06-01.html"}`
（周报存 `data/weekly/`，月报存 `data/monthly/`）

上传后：后台 http://13.214.205.219/airwallex/admin.html → 对应「周报审核 / 月报审核」出现**草稿**，可预览、可点「通过发布」。

> 旧端点 `/api/weekly` 仍可用（等价于 `type=weekly`），但建议统一改用 `/api/reports`。

---

## 2. ⚠️ HTML 资产路径（重要约束）

上传的报告会被服务在 **`/airwallex/data/<type>/<period>.html`**（周报 `data/weekly/`、月报 `data/monthly/`）。所以页面里引用平台样式必须用**绝对路径**，不能用相对路径：

```html
<!-- ✅ 对 -->
<link rel="stylesheet" href="/airwallex/assets/style.css">
<!-- ❌ 错（会 404，因为相对到了 data/weekly/assets/）-->
<link rel="stylesheet" href="assets/style.css">
```

或者把 CSS/JS **内联**进 HTML，做成完全自包含的单文件（更稳）。
（仓库里现成的 `web/weekly.html` 用的是相对路径，迁过来上传前需把资产路径改成绝对路径或内联。）

## 3. （可选）直接发布 / 查状态

- **发布**（一般由后台点按钮触发，也可调用，无需 token）：
  ```
  POST /api/reports/publish   JSON 或 form: {"period":"2026-06-01","type":"monthly"}
  ```
- **查某类型所有周期 slots 及状态**：
  ```
  GET /api/reports?type=weekly        （或 type=monthly）
  → {"type":"weekly","slots":[{"period":"2026-06-22","state":"empty|draft|published","title":"","url":...}, ...]}
  ```

---

## 4. 约束 / 注意
- 同一 `period` 重复上传 = 覆盖该周草稿（状态重置为 draft）。
- `period` 必须是 `YYYY-MM-DD`；落在 6–7 月周一之外的也能传，但不会在当前 slots 列表里显示。
- 上传走 token；发布暂不需要（demo 简化，生产应把后台放登录后）。
- 媒体（AI 播客/视频）目前为占位；对外前请复核文案与数字真伪。
